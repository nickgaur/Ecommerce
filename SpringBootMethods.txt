4 types of framework to interact with Database
-------------------------------------------------
	a. JDBC
		i. Need to Write more SQL Queries
		ii. And write a lot of java code.
	b. Spring JDBC
		i. Write a lot of SQL Queries
		ii. But Lesser java code

	c. JPA
		i. Do not worry about queries.
		ii. Just Map Entities to Tables.

	d. Spring Data JPA
		i. Let's make JPA even more simple.
		ii. I will take care of everything.

@RequestMapping("")
--------------------
	@RequestMapping (Class Level) => this annotation is used for mapping all the request starting with the value passed in it. E.g. @RequestMapping("/api/v1"). It can be treated as a base Route.
	@RequestMapping (Method Level) => This is used to define the method and endpoint. E.g. @RequestMapping(value="/api/categories", method=RequestMethod.GET)

In Memory Authentication
--------------------------
In-memory authentication in Spring Security is a method of storing user credentials (username, password, roles) directly in the application's volatile memory rather than a database. It is primarily designed for development, testing, and prototyping because user details are lost when the server restarts.

{noop} Prefix
--------------
In Spring Security, (noop} refers to a special prefix used to designate the NoOpPasswordEncoder, a password encoder that does not perform any actual password encoding or hashing. It is provided for legacy and testing purposes only and is highly insecure for production environments.


JwtUtils
----------
Contains utility method for generating, parsing, and validating JWTs.
Include Generating a token from a username, validating a JWT, and extracting the username from a token.

AuthTokenFilter
----------------
Filters incoming requests to check for a valid JWT in the header, Setting the authentication context if the token is valid.
Extracts JWT from a request header, validates it, and configures the Spring Security context with user details if the token is valid.

AuthEntryPointJwt
------------------
Provides custom handling for uauthorized requests, typically when authentication is required but not supplied or valid.
When an unauthorized request is detected, it logs the error and returns a JSON response with an error message, status code, and the path attempted.

SecurityConfig
---------------
Configures Spring Security filters and rules for the application.
Sets Up the security filter chain, permitting or denying access based on paths and roles. It also configures session management to stateless, which is crucial for JWT usage.
